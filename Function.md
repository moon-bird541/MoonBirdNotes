# 功能说明

## 1. 登录

系统当前仅支持管理员本人登录使用，不开放用户注册。用户登录成功后，通过当前统一的认证 Provider 完成身份校验，并访问系统中的笔记与管理功能。

## 2. 上传 Markdown 笔记

系统支持上传 Markdown（`.md`）文件作为笔记。上传后会保留原始 Markdown 内容，并读取正文用于后续展示、编辑和版本管理。

## 3. 标签规则

用户在上传或编辑笔记时必须填写标签。每篇笔记最多支持两个标签。标签既可以选择系统中已有标签，也可以在输入时直接创建新标签。

## 4. 笔记列表展示

系统支持分页展示笔记列表，默认按创建时间倒序排列。列表中会显示笔记标题、摘要、字数、时间信息，并在标题下方展示轻量胶囊样式标签，方便快速浏览和定位内容。

## 5. 笔记详情查看

系统支持查看单篇笔记详情。详情页会展示笔记标题、标签、字数、原始文件信息、创建时间、更新时间，以及 Markdown 渲染后的正文内容。当前已补充统一的 Markdown 渲染样式，使标题、列表、引用、代码块等内容更稳定易读。

## 6. 在线编辑笔记

系统支持在线编辑笔记的标题、正文和标签。编辑页当前采用所见即所得模式，用户在编辑时无需直接面对 Markdown 原文，但保存时仍然写回纯净 Markdown 内容，不引入内嵌 HTML。

编辑器当前支持：
- 选中文字后显示悬浮工具条
- 标题等级设置
- 加粗
- 斜体
- 删除线
- 行内代码
- 引用
- 普通代码块
- 按钮悬停提示功能说明
- 代码块语言标签原位编辑

保存后会直接覆盖当前笔记内容，并重新渲染 HTML，同时刷新更新时间。回收站中的笔记不允许直接编辑，必须先恢复后再进行修改。

## 7. 历史版本与回退

系统会在用户实际修改笔记标题、正文或标签后，自动保存修改前的内容为历史版本，便于后续回退。每篇笔记最多保留最近 5 个历史版本。

如果用户保存时没有实际修改标题、正文或标签，则不会生成新的历史版本，也不会刷新更新时间。用户可以在编辑页查看历史版本，并将当前笔记回退到指定版本。

## 8. 删除与回收站

系统当前采用软删除机制。用户删除笔记后，笔记会先进入回收站，而不是立刻从数据库中移除。在回收站中，用户可以恢复笔记，也可以执行彻底删除。被删除的笔记默认保留 30 天，超过保留期后系统会自动清理。

## 9. 页面结构

系统当前已提供以下主要页面：

- 登录页
- 首页
- 笔记列表页
- 笔记上传页
- 笔记详情页
- 笔记编辑页
- 回收站页

左侧功能栏支持收起与展开，便于在不同页面之间快速切换。
