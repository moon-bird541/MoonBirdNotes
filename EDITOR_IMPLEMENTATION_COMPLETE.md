## 🔧 技术实现要点

### 1. CSS 变量系统
使用 CSS 变量定义设计 Token，便于后续主题切换扩展：
```css
:root {
  --editor-bg-dark: #0f1419;
  --editor-accent: #f59e0b;
  /* ... */
}
```

### 2. 样式隔离
新建独立的 `editor-dark.css` 文件，使用 `!important` 覆盖原有样式，避免直接修改导致的样式冲突。

### 3. 深度选择器
使用 `:deep()` 选择器修改第三方组件（Toast UI Editor、Element Plus）的内部样式。

### 4. backdrop-filter
使用 `backdrop-filter: blur()` 实现半透明模糊效果，提升现代感。

## 🎯 设计原则

### 遵循的 UX 最佳实践

1. **对比度** ✅
   - 文本对比度 ≥ 4.5:1（符合 WCAG AA 标准）
   - 主要文本：白色 `#e4e6eb` vs 深色背景

2. **可读性** ✅
   - 行高 `1.8`（舒适阅读）
   - 内容宽度 `820px`（60-75 字符/行）
   - 等宽字体提升代码可读性

3. **交互反馈** ✅
   - 聚焦时琥珀色边框
   - 按钮悬停状态明确
   - 加载和保存状态清晰

4. **响应式设计** ✅
   - 移动端优化布局
   - 字号自适应
   - 触摸目标 ≥ 44px

5. **性能** ✅
   - 使用 `transform` 和 `opacity` 实现动画
